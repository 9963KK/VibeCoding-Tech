# /JVibe:status - 查看项目状态

你需要快速展示项目的功能状态、进度统计和最近活跃情况。

## 任务

调用 **doc-sync Sub-Agent** 完成以下工作：

1. **读取功能清单**
   - 统计各状态功能数量（✅/🚧/❌）
   - 计算总体完成率

2. **按模块统计**
   - 从项目文档读取模块列表
   - 统计每个模块的功能数量和完成情况

3. **识别关键信息**
   - 最近开发中的功能
   - 最近完成的功能
   - 长期未开始的功能（如有）

## 输出格式

```
========================================
  📊 JVibe 项目状态
========================================

项目：{项目名称}
更新时间：{当前日期}

----------------------------------------
功能总览
----------------------------------------
  总功能数: {总数}
  ✅ 已完成: {已完成数} ({完成率}%)
  🚧 开发中: {开发中数}
  ❌ 未开始: {未开始数}

----------------------------------------
模块统计
----------------------------------------
  AuthModule
    总数: 5  |  ✅ 5  |  🚧 0  |  ❌ 0  |  完成率: 100%

  UserModule
    总数: 5  |  ✅ 5  |  🚧 0  |  ❌ 0  |  完成率: 100%

  ChatModule
    总数: 10  |  ✅ 7  |  🚧  2  |  ❌ 1  |  完成率: 70%

----------------------------------------
🔨 开发中的功能
----------------------------------------
  🚧 F-018: 文件上传
  🚧 F-019: 消息撤回

----------------------------------------
✅ 最近完成
----------------------------------------
  ✅ F-015: 用户头像上传
  ✅ F-016: 在线状态显示
  ✅ F-017: 消息已读回执

========================================
```

## 可选：按优先级分组

如果功能清单中标注了优先级（P0/P1/P2），可以额外展示：

```
----------------------------------------
⚡ 高优先级功能 (P0)
----------------------------------------
  ✅ F-001: 用户注册
  ✅ F-002: 用户登录
  🚧 F-018: 文件上传
  ❌ F-020: 消息加密
```

## 可选：显示阻塞信息

如果某些功能有依赖关系或被阻塞，可以提示：

```
----------------------------------------
⚠️  需要注意
----------------------------------------
  - F-020 (消息加密) 依赖 F-018 (文件上传) 完成
  - F-021 (消息撤回) 已开发中超过 2 周
```

## 调用方式

```yaml
# 主 Agent 调用 doc-sync Sub-Agent
task: 生成项目状态报告
agent: doc-sync
input:
  - 功能清单
  - 项目文档
output:
  - 格式化的状态报告
```

## 简洁模式

如果用户只需要简单统计，可以输出简洁版本：

```
📊 项目状态
  总数: 20  |  ✅ 18 (90%)  |  🚧 1  |  ❌ 1

🚧 开发中: F-018, F-019
✅ 最近完成: F-015, F-016, F-017
```

## 详细模式

如果用户需要详细信息，可以列出每个功能的 TODO 完成情况：

```
========================================
  📋 详细功能状态
========================================

ChatModule

  ✅ F-015: 用户头像上传
    ✅ 8/8 TODO 已完成

  🚧 F-018: 文件上传
    ✅ 6/9 TODO 已完成 (67%)
    - ✅ 实现 POST /api/files 端点
    - ✅ 文件类型验证
    - ✅ 文件大小验证
    - ✅ 图片缩略图生成
    - ✅ S3 云存储集成
    - ✅ 上传进度反馈
    - ⏳ 单元测试
    - ⏳ 集成测试
    - ⏳ API文档更新

  ❌ F-020: 消息加密
    ⏳ 0/7 TODO 未开始
```

## 注意事项

1. **数据准确**：统计数据必须与功能清单完全一致
2. **格式美观**：使用表格或对齐的文本，易于阅读
3. **重点突出**：用 emoji 和颜色（如果支持）突出关键信息
4. **快速响应**：使用 haiku 模型，确保快速返回
5. **可配置**：根据项目规模选择简洁或详细模式
